# DeepResearch Platform 架构文档

## 系统概述

DeepResearch Platform 是一个基于 ReAct (Reasoning + Acting) 模式的 AI 驱动深度研究平台。系统采用前后端分离架构，通过多阶段迭代工作流实现自动化研究分析。

## 多阶段工作流详解

### 工作流概述

系统采用 ReAct (Reasoning + Acting) 模式的迭代工作流，共包含 6 个主要阶段：

```
┌─────────────────────────────────────────────────────────────────┐
│                      研究工作流 (Research Workflow)               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐                │
│  │  Stage 1 │     │  Stage 2 │     │  Stage 3 │                │
│  │ Planning │ --> │ Searching│ --> │  Search  │                │
│  │  (规划)  │     │  (搜索)  │     │ Analysis │                │
│  │          │     │          │     │  (分析)  │                │
│  └──────────┘     └──────────┘     └────┬─────┘                │
│                                         │                       │
│                    ┌────────────────────┘                       │
│                    │ (need_more_search = true?)                  │
│                    ▼                                             │
│              ┌──────────┐                                       │
│              │ Iteration│ (最多 2 轮)                            │
│              │  (迭代)  │                                       │
│              └────┬─────┘                                       │
│                   │                                              │
│                   └────────────────────┐                        │
│                                        ▼                        │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐                │
```

### 各阶段详细说明

#### Stage 1: Planning (规划阶段)

**输入**: 用户查询  
**输出**: 搜索查询列表 (3-5 个)

**流程**:
1. LLM 分析用户查询的意图和关键概念
2. 生成覆盖不同角度的搜索子查询
3. 制定研究策略

**提示词**: `planning_prompt.txt`

**示例**:
```
输入: "2024年人工智能芯片市场发展趋势"
输出: [
  "2024年人工智能芯片市场发展趋势",
  "AI芯片市场规模预测 2024",
  "NVIDIA AMD Intel AI芯片竞争 2024",
  "中国AI芯片产业发展现状 2024"
]
```

#### Stage 2: Searching (搜索阶段)

**输入**: 搜索查询列表  
**输出**: 搜索结果集合

**流程**:
1. 并行执行多个 DuckDuckGo 搜索
2. 收集每个查询的搜索结果
3. 去重和排序
4. 发送 tool_call/tool_result 事件到前端

**工具**: `search_web` (DuckDuckGo)

#### Stage 3: Search Analysis (搜索分析阶段)

**输入**: 搜索结果 (前 15 条)  
**输出**: 分析结果 JSON

**流程**:
1. LLM 评估搜索结果质量
2. 提取关键信息点
3. 识别信息缺口
4. 决定是否需要补充搜索
5. 如需要，生成新的搜索查询

**提示词**: `search_analysis_prompt.txt`

**输出格式**:
```json
{
  "analysis_summary": "搜索结果总体评估...",
  "need_more_search": true/false,
  "additional_queries": ["补充查询1", "补充查询2"],
  "information_gaps": ["信息缺口1"],
  "reasoning": "决策理由..."
}
```

#### Stage 4: Scraping (内容抓取阶段)

**输入**: 去重后的搜索结果 (前 5 个)  
**输出**: 详细内容

**流程**:
1. 抓取前 5 个关键来源的完整内容
2. 使用 Jina AI 提取网页正文
3. 存储内容供后续分析使用

**工具**: `scrape_url` (Jina AI)

#### Stage 5: Deep Analysis (深度分析阶段)

**输入**: 所有搜索结果 (前 20 条)  
**输出**: 深度分析 JSON

**流程**:
1. 综合分析所有收集的信息
2. 识别关键主题和模式
3. 分析不同观点间的关系
4. 评估证据质量
5. 形成深度洞察

**提示词**: `deep_analysis_prompt.txt`

**输出格式**:
```json
{
  "executive_summary": "执行摘要...",
  "key_themes": [...],
  "key_insights": [...],
  "perspective_analysis": [...],
  "trends": [...],
  "evidence_quality": {...}
}
```

#### Stage 6: Synthesis (综合总结阶段)

**输入**: 抓取的内容 + 深度分析结果  
**输出**: 结构化研究报告

**流程**:
1. 基于深度分析洞察生成报告
2. 结构化输出 (7 个部分)
3. 每个论断标注来源引用
4. 确保报告字数 >= 3000 字

**提示词**: `synthesis_prompt.txt`

**输出结构**:
- 📌 核心摘要
- 🔍 研究路径
- 💡 关键发现
- ⚖️ 多维分析
- ❓ 未解问题
- 📚 参考文献

## 组件说明

### 前端组件

| 组件 | 文件 | 职责 |
|------|------|------|
| ResearchChat | `ResearchChat.tsx` | 用户输入区，支持超长文本 (50,000 字符)，自动调整高度 |
| ReportViewer | `ReportViewer.tsx` | 报告展示，卡片式布局，Markdown 渲染 |
| ConsolePanel | `ConsolePanel.tsx` | 实时控制台，显示工具调用和分析摘要 |
| ProgressBar | `ProgressBar.tsx` | 进度条，显示当前阶段和百分比 |
| SettingsPanel | `SettingsPanel.tsx` | 设置面板，API Key 和模型选择 |

### 后端模块

| 模块 | 文件 | 职责 |
|------|------|------|
| ResearchWorkflow | `workflow.py` | 多阶段工作流编排，ReAct 迭代逻辑 |
| LLM | `llm.py` | LLM 配置，SiliconFlow API 封装 |
| Search | `search.py` | 搜索工具，DuckDuckGo 集成 |
| Scrape | `scrape.py` | 网页抓取，Jina AI 集成 |
| PDF | `pdf.py` | PDF 分析，PyMuPDF 集成 |

## 扩展性设计

### 添加新工具

1. 在 `backend/tools/` 创建新工具文件
2. 实现工具函数，返回标准格式
3. 在 `workflow.py` 中调用

### 添加新阶段

1. 在 `workflow.py` 的 `execute` 方法中添加阶段
2. 创建对应的提示词文件
3. 实现阶段处理方法

### 添加新模型

1. 在 `backend/agent/llm.py` 添加模型配置
2. 前端设置面板自动同步

## 性能考虑

- **并发搜索**: 多个查询并行执行
- **SSE 流式**: 实时反馈，减少等待焦虑
- **迭代限制**: 最多 2 轮迭代，避免过度搜索
- **内容截断**: LLM 输入限制处理
- **超时控制**: 工具调用和总流程超时

## 安全设计

- **API Key 本地存储**: 不发送到后端
- **XSS 过滤**: 用户输入消毒
- **URL 校验**: 只允许合法 URL
- **CORS 配置**: 限制跨域访问
