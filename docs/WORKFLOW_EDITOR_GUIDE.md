# 进阶工作流编辑器使用指南

## 概述

进阶工作流编辑器是 DeepResearch Platform 的专业级可视化工作流设计工具，采用 ComfyUI 风格的节点编辑器界面，支持复杂研究工作流的创建、配置和执行。

## 功能特性

### 1. 可视化节点编辑
- **拖拽式操作**：从左侧节点面板拖拽节点到画布
- **节点连接**：拖拽连接点创建节点间的数据流
- **实时配置**：双击节点或展开详情进行配置
- **模板系统**：提供预设工作流模板快速开始

### 2. 实时执行状态可视化
- **节点状态动画**：
  - 🔵 **蓝色脉冲** = 正在执行
  - 🟢 **绿色边框** = 执行成功
  - 🔴 **红色边框** = 执行失败
  - ⚪ **灰色边框** = 待执行
- **进度指示**：节点显示执行进度百分比
- **执行时间**：显示每个节点的执行耗时

### 3. 错误定位与处理
- **错误标识**：失败节点显示红色闪烁边框和错误徽章
- **错误详情**：点击错误徽章查看详细错误信息
- **解决方案**：错误弹窗提供建议解决方案
- **快速重试**：支持单个节点重试执行

### 4. 日志展示系统
- **实时日志**：展开节点查看执行日志
- **日志级别**：不同级别用不同颜色标识
  - 🔵 `[DEBUG]` - 调试信息（蓝色）
  - ⚪ `[INFO]` - 普通信息（灰色）
  - 🟡 `[WARN]` - 警告信息（黄色）
  - 🔴 `[ERROR]` - 错误信息（红色）
- **日志搜索**：支持在右侧面板搜索日志内容

### 5. 执行路径可视化
- **已执行路径**：绿色实线显示
- **当前执行路径**：蓝色流动动画
- **待执行路径**：灰色虚线显示
- **连接点状态**：根据节点状态变色

## 快速开始

### 1. 进入进阶模式

1. 在应用顶部导航栏点击"进阶模式"切换按钮
2. 系统会自动加载工作流编辑器界面

### 2. 创建工作流

#### 方式一：从模板开始（推荐）
1. 点击画布中央的"选择模板"按钮
2. 在弹出的模板选择器中浏览可用模板：
   - **标准研究流程**：规划 → 搜索 → 分析 → 综合
   - **迭代研究流程**：带质量检查的迭代执行
   - **快速搜索**：简化的搜索流程
3. 选择模板后点击"应用模板"

#### 方式二：从头创建
1. 从左侧节点面板拖拽节点到画布
2. 拖拽节点的输出连接点到另一个节点的输入连接点
3. 重复步骤直到完成工作流设计

### 3. 配置节点

#### 基础配置
1. **双击节点名称**：直接编辑节点名称
2. **点击描述区域**：添加或编辑节点描述
3. **展开详情**：点击"展开详情"查看和编辑更多参数

#### 详细配置
1. 点击节点选中它
2. 在右侧面板的"基础"选项卡中配置：
   - 节点名称
   - 节点描述
   - 标签
3. 在"业务"选项卡中配置执行参数：
   - 超时时间
   - 重试次数
   - 重试策略
   - 节点专属参数（如查询数、结果数等）
4. 在"高级"选项卡中配置：
   - 日志级别
   - 优先级
   - 并行执行开关

### 4. 执行工作流

#### 开始执行
1. 点击顶部"开始执行"按钮
2. 在弹出的对话框中输入研究主题/查询内容
3. 查看执行配置摘要
4. 点击"开始执行"

#### 监控执行
- **整体进度**：查看顶部进度条和统计信息
- **当前节点**：查看正在执行的节点名称
- **节点状态**：观察节点边框颜色和动画
- **执行日志**：展开节点查看实时日志

#### 处理错误
1. 如果节点执行失败，会显示红色边框和错误徽章
2. 点击错误徽章查看错误详情
3. 根据建议解决方案修改配置
4. 点击"重试执行"或"修改配置"

### 5. 保存和加载

#### 保存工作流
- 点击工具栏"保存"按钮
- 或使用快捷键 `Ctrl+S`

#### 加载工作流
- 点击工具栏"打开"按钮
- 选择已保存的工作流文件

## 节点类型说明

### 流程控制节点

#### 开始节点 (Start)
- **功能**：工作流入口
- **输入**：无
- **输出**：1个
- **用途**：标记工作流开始位置

#### 结束节点 (End)
- **功能**：工作流出口
- **输入**：1个
- **输出**：无
- **用途**：标记工作流结束位置

#### 条件节点 (Condition)
- **功能**：条件分支控制
- **输入**：1个
- **输出**：2个（真/假分支）
- **配置**：条件表达式、分支标签

#### 循环节点 (Loop)
- **功能**：迭代执行控制
- **输入**：1个
- **输出**：2个（继续/退出）
- **配置**：最大迭代次数、退出条件

### 研究节点

#### 规划节点 (Planning)
- **功能**：生成搜索查询策略
- **配置**：
  - 最大查询数：1-20
  - 使用增强规划：开关
  - 领域检测：开关

#### 搜索节点 (Search)
- **功能**：执行网络搜索
- **配置**：
  - 搜索引擎：UAPI/Bing/Baidu/DuckDuckGo
  - 最大结果数：1-50
  - 时间范围：1-365天
  - 安全搜索：开关

#### 分析节点 (Analysis)
- **功能**：分析搜索结果质量
- **配置**：
  - 启用质量评估：开关
  - 质量阈值：0-100
  - 分析深度：基础/标准/深度

#### 综合节点 (Synthesis)
- **功能**：生成最终研究报告
- **配置**：
  - 报告格式：结构化/Markdown/学术/执行摘要
  - 最大长度：100-50000字符
  - 包含来源：开关

## 最佳实践

### 1. 工作流设计
- **保持简洁**：一个工作流专注于一个研究主题
- **合理命名**：为节点和连接添加描述性名称
- **使用模板**：从预设模板开始，根据需求调整

### 2. 错误处理
- **设置超时**：为每个节点设置合理的超时时间
- **配置重试**：对关键节点启用重试机制
- **检查日志**：执行失败时查看详细日志

### 3. 性能优化
- **并行执行**：对独立的节点启用并行执行
- **限制迭代**：设置合理的最大迭代次数
- **优化查询**：在规划节点中优化搜索查询

## 故障排除

### 节点无法连接
- 检查源节点的输出类型是否与目标节点的输入类型匹配
- 确保没有创建循环连接
- 检查节点是否已达到最大连接数

### 执行失败
- 查看节点日志获取详细错误信息
- 检查网络连接状态
- 验证 API 密钥是否有效
- 增加超时时间

### 界面卡顿
- 减少同时显示的日志数量
- 关闭不必要的节点详情展开
- 使用 MiniMap 快速导航
