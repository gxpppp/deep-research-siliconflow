## 实现目标
使后端能够将内容实时流式传输到前端界面右侧的过程可视化栏目中，模拟网页版AI助手（如DeepSeek和豆包）的逐字逐句输出效果。

## 当前架构分析
- 后端与SiliconFlow API：使用`ainvoke`（非流式，一次性返回）
- 后端与前端：使用SSE（Server-Sent Events）流式推送
- 前端展示：目前只推送状态更新，不展示LLM生成的内容流

## 实现方案

### 阶段一：后端改造 - 添加流式LLM支持

#### 1.1 修改 `backend/agent/llm.py`
- 添加`create_streaming_llm`函数，配置`streaming=True`
- 保留原有`create_llm`用于非流式场景

#### 1.2 修改 `backend/agent/workflow.py`
- 在关键阶段（规划、分析、深度分析、综合）使用流式输出
- 新增SSE事件类型：`content_chunk`（内容片段）、`content_complete`（内容完成）
- 实现`astream`调用，实时yield内容块到前端

#### 1.3 新增流式处理辅助函数
- `stream_llm_response()`：处理流式响应，yield内容块
- 支持累积缓冲区，处理JSON/文本混合输出

### 阶段二：前端类型定义扩展

#### 2.1 修改 `frontend/src/types/index.ts`
- 新增`StreamingContent`类型：
  ```typescript
  export interface StreamingContent {
    id: string
    stage: 'planning' | 'search_analysis' | 'deep_analysis' | 'synthesis'
    title: string
    content: string  // 当前累积的完整内容
    isStreaming: boolean
    timestamp: Date
  }
  ```
- 扩展`ProcessData`：添加`streamingContents: StreamingContent[]`
- 新增SSE事件类型：`content_chunk` | `content_complete`

### 阶段三：前端状态管理扩展

#### 3.1 修改 `frontend/src/stores/researchStore.ts`
- 添加流式内容管理方法：
  - `startStreamingContent(stage, title)`：开始新的流式内容
  - `appendStreamingContent(id, chunk)`：追加内容片段
  - `completeStreamingContent(id)`：标记流式完成
  - `clearStreamingContents()`：清空流式内容

### 阶段四：流式展示组件开发

#### 4.1 新建 `frontend/src/components/StreamingPanel.tsx`
- 实时展示流式输出的内容
- 支持Markdown渲染（代码块、列表、表格等）
- 打字机效果：内容逐字/逐句显示
- 光标闪烁动画表示正在生成
- 自动滚动到最新内容

#### 4.2 新建 `frontend/src/components/StreamingContentCard.tsx`
- 单条流式内容的卡片组件
- 显示阶段标签（规划/分析/综合等）
- 内容区域支持语法高亮
- 完成后的收起/展开功能

#### 4.3 修改 `frontend/src/components/ProcessVisualizer.tsx`
- 新增"实时输出"Tab
- 整合StreamingPanel到Tab界面
- 自动切换到有新内容的Tab

### 阶段五：API层和事件处理

#### 5.1 修改 `frontend/src/services/api.ts`
- 已支持SSE，无需修改

#### 5.2 修改 `frontend/src/components/ResearchChat.tsx`
- 在SSE事件处理器中添加：
  - `content_chunk`：调用`appendStreamingContent`
  - `content_complete`：调用`completeStreamingContent`
- 各阶段开始时调用`startStreamingContent`

### 阶段六：打字机效果实现

#### 6.1 新建 `frontend/src/hooks/useTypewriter.ts`
- 自定义Hook实现打字机效果
- 参数：目标文本、打字速度、是否启用
- 返回：当前显示文本、是否完成

#### 6.2 或直接使用CSS动画
- 使用`animation`实现光标闪烁
- 内容直接显示（更流畅，类似ChatGPT）
